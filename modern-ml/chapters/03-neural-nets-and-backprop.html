Chapter III

Neural Networks and Backpropagation

A neural network is just function composition. Pile up alternating linear maps and nonlinearities; differentiate the whole thing by the chain rule; minimize the loss by gradient descent. The depth of the consequences is not obvious from the simplicity of the recipe.

Before we start

A neural network is a stack of two kinds of operation, repeated: a linear map (multiply by a matrix, add a bias) and an activation (a fixed nonlinear function applied to each entry). To train it, you compute how much the loss changes when you nudge each weight — the gradient — then move every weight a tiny step downhill. Repeat for many billions of examples.

The clever algorithm that makes computing those gradients tractable is backpropagation, which is just the calculus chain rule run through the network in reverse. Modern frameworks (PyTorch, JAX) implement backprop automatically; your job is to write the forward pass and they hand you the gradient. The rest of the chapter covers the engineering that makes this train reliably at scale: smarter optimizers (SGD, Adam, AdamW), careful weight initialization, and normalization layers.

Terms you'll meet

MLP (multilayer perceptron)
The basic neural network: alternating fully-connected linear maps and elementwise nonlinearities.
Layer
One linear map plus its nonlinearity. A "10-layer network" has ten of them stacked.
Weights and bias
The trainable numbers inside a layer. Weights form a matrix; the bias is an added offset vector.
Hidden unit / hidden activation
One of the intermediate values inside the network — the output of one neuron in one layer.
Activation function
The fixed nonlinearity applied entry-wise after each linear map. Without it the network collapses to a single big linear map.
ReLU
$\max(0, z)$. Cheap, surprisingly effective, dominant since 2010.
GELU
A smooth variant of ReLU. The default activation in transformer feed-forward layers.
Sigmoid / tanh
The 1990s defaults. Saturate at large inputs (gradients shrink to zero), which made training deep networks hard.
Vanishing / exploding gradients
When gradients through many layers shrink to zero (vanishing) or blow up (exploding), training breaks. The reason for most architectural tricks in this chapter.
Universal approximation
The theoretical guarantee that a wide enough MLP can approximate any continuous function. Says nothing about whether training will actually find it.
Forward pass
Running an input through the network to get a prediction.
Backward pass
Running the gradient backward through the network to compute, for every weight, "how much would the loss change if I nudged this weight."
Backpropagation
The algorithm for the backward pass. Mechanically applies the chain rule to every operation in the network, in reverse order.
Autodiff (automatic differentiation)
The general technique backprop is an instance of. Frameworks implement it once, so you never write a backward pass by hand.
Computation graph
The bookkeeping structure that records every operation in the forward pass so the framework knows how to run it in reverse.
Gradient descent
The training algorithm: compute the gradient of the loss, subtract a small fraction of it from the parameters, repeat.
SGD (stochastic gradient descent)
Gradient descent but computed on a small random subset of the data (a "minibatch") at each step. Cheaper, and the noise actually helps.
Minibatch
The handful (8 to 8000) of examples used to estimate the gradient at each step.
Learning rate $\eta$
How big a step to take. The single most important hyperparameter.
Epoch
One full pass through the training set.
Momentum
An exponentially-weighted average of past gradients added to the update. Accelerates progress along consistent directions.
Adam / AdamW
The modern default optimizers. Track running averages of the gradient and its square, giving each parameter its own adaptive step size. AdamW is the variant with cleaner weight decay; standard for transformer training.
Weight decay
A penalty that gently shrinks weights toward zero each step — neural-net regularization, mathematically similar to ridge.
Learning-rate schedule
How $\eta$ changes through training. Typically: warmup (ramp up over the first few hundred steps), then cosine decay back down to near zero.
Initialization
How the weights are set before training begins. Glorot and Kaiming initializations pick the right initial variance so signals don't vanish or explode through depth.
Normalization layer
A built-in layer that re-scales activations to a standard distribution every forward pass. BatchNorm normalizes across the batch (used in CNNs); LayerNorm normalizes across features for each example (used in transformers); RMSNorm is a cheaper LayerNorm without the mean-subtraction step.
Over-parameterization
Having far more parameters than training examples. Counter-intuitively, large nets do this on purpose — and generalize anyway.
Implicit regularization
The way the optimizer itself prefers certain solutions among the many that fit training data — a phenomenon nobody fully understands but that explains why huge nets work.
Gradient clipping
If the gradient gets very large, scale it down to a fixed maximum norm. Cheap insurance against blowups.

§1The multilayer perceptron

An MLP with $L$ layers is a sequence of affine maps interleaved with elementwise nonlinearities:

$$ \mathbf{h}_0 = \mathbf{x}, \qquad \mathbf{h}_\ell = \sigma\bigl( \mathbf{W}_\ell\, \mathbf{h}_{\ell - 1} + \mathbf{b}_\ell \bigr), \qquad f_\theta(\mathbf{x}) = \mathbf{W}_L\, \mathbf{h}_{L-1} + \mathbf{b}_L. $$

Where

$\mathbf{W}_\ell \in \mathbb{R}^{d_\ell \times d_{\ell-1}}$
Layer $\ell$ weight matrix; one row per output unit, one column per input.
$\mathbf{b}_\ell \in \mathbb{R}^{d_\ell}$
Bias vector.
$\sigma$
Elementwise nonlinearity (ReLU, GELU, …). Without it, the whole stack collapses to a single linear map.
$\mathbf{h}_\ell$
Hidden activations at layer $\ell$. The last layer skips $\sigma$ for regression, or pushes through softmax for classification.
$\boldsymbol{\theta} = \{\mathbf{W}_\ell, \mathbf{b}_\ell\}_\ell$
All parameters, lumped.

The universal approximation theorems say a wide enough MLP can approximate any continuous function. They do not say how wide, or that training will find such an approximator, or that the result will generalize. Approximation is necessary but not sufficient. Everything interesting in deep learning lies in the gaps between "exists" and "trainable" and "generalizes."

§2Activation functions

The 1990s ran on $\tanh$ and the logistic sigmoid. Both saturate for large inputs, so gradients shrink to zero through deep stacks — the vanishing gradient problem. The 2010s fix was the rectified linear unit,

$$ \mathrm{ReLU}(z) \;=\; \max(0, z), $$

which has gradient 1 on its positive half and 0 on its negative half. Cheap, non-saturating in the active region, and surprisingly hard to improve on (Nair & Hinton 2010). Its dead-neuron failure mode — a unit stuck at zero stays there because gradient is zero — motivated Leaky ReLU, ELU, and the variants below.

The modern default in transformers is GELU (Gaussian Error Linear Unit; Hendrycks & Gimpel 2016),

$$ \mathrm{GELU}(z) \;=\; z \cdot \Phi(z) \;\approx\; \half z\Bigl(1 + \tanh\bigl(\sqrt{2/\pi}\,(z + 0.044715\, z^3)\bigr)\Bigr), $$

Where

$z$
A scalar — one pre-activation value (the output of a linear map, before the nonlinearity). The function is applied to every entry independently.
$\Phi(z)$
A scalar in $(0,1)$ — the standard-normal cumulative distribution function: the probability a unit Gaussian lands below $z$. It plays the role of a soft, input-dependent gate.
$\tanh$
The hyperbolic tangent, an S-shaped function from $-1$ to $1$; used here only to approximate $\Phi$ cheaply.

where $\Phi$ is the standard normal CDF. GELU smoothly gates by the input's percentile rather than a hard cut at zero. SwiGLU (Shazeer 2020) is the gated-linear variant that dominates current LLM feedforward layers; we will meet it in Chapter V.

§3Backpropagation is reverse-mode autodiff

Training requires the gradient $\nabla_{\boldsymbol{\theta}} \mathcal{L}$. Computing it by symbolic differentiation is intractable for large networks; computing it by finite differences is hopelessly noisy and slow. The right answer is to apply the chain rule mechanically in a particular order.

A neural network is a directed acyclic computation graph. Forward mode walks the graph from inputs to outputs computing values and propagating directional derivatives — efficient when there are few inputs and many outputs. Reverse mode walks forward to record activations, then walks backward propagating gradients with respect to each output against every intermediate. Reverse mode is efficient when there are few outputs (a scalar loss) and many parameters — exactly our situation.

For a single layer $\mathbf{h}_\ell = \sigma(\mathbf{z}_\ell)$ with $\mathbf{z}_\ell = \mathbf{W}_\ell \mathbf{h}_{\ell-1} + \mathbf{b}_\ell$, the local backward rules are

$$ \frac{\partial \mathcal{L}}{\partial \mathbf{z}_\ell} \;=\; \frac{\partial \mathcal{L}}{\partial \mathbf{h}_\ell} \odot \sigma'(\mathbf{z}_\ell), \qquad \frac{\partial \mathcal{L}}{\partial \mathbf{W}_\ell} \;=\; \frac{\partial \mathcal{L}}{\partial \mathbf{z}_\ell}\, \mathbf{h}_{\ell-1}^\top, \qquad \frac{\partial \mathcal{L}}{\partial \mathbf{h}_{\ell-1}} \;=\; \mathbf{W}_\ell^\top\, \frac{\partial \mathcal{L}}{\partial \mathbf{z}_\ell}. $$

Where

$\mathbf{z}_\ell$
A vector — the pre-activation at layer $\ell$ ($\mathbf{z}_\ell = \mathbf{W}_\ell \mathbf{h}_{\ell-1} + \mathbf{b}_\ell$), before $\sigma$ is applied.
$\partial \mathcal{L} / \partial \mathbf{h}_\ell$
A vector of the same shape as $\mathbf{h}_\ell$ — how much the loss changes per unit change in each activation. This is the signal handed down from the layer above.
$\sigma'(\mathbf{z}_\ell)$
A vector — the derivative of the activation, evaluated entry-by-entry. For ReLU it is $1$ where $z>0$ and $0$ otherwise.
$\odot$
The Hadamard (elementwise) product: multiply two equal-length vectors entry by entry, giving a vector — not a dot product.
$\mathbf{h}_{\ell-1}^\top$
The transpose of the previous layer's activations, turning the column vector into a row so the product $\frac{\partial\mathcal{L}}{\partial\mathbf{z}_\ell}\,\mathbf{h}_{\ell-1}^\top$ is a matrix the same shape as $\mathbf{W}_\ell$ (an outer product).
$\mathbf{W}_\ell^\top$
The transpose of the weight matrix. Multiplying by it sends the gradient backward across the layer — the mirror image of the forward map.

That's it. The whole of backpropagation is the chain rule applied to each operation in the graph, plus the bookkeeping to traverse the graph in topological-reverse order. Total cost is the same as a forward pass within a constant factor (Baur & Strassen 1983 — the cheap gradient principle).

What "PyTorch" actually does for you

You don't write backward passes anymore. Modern frameworks (PyTorch, JAX, MLX) build the computation graph at runtime or trace it once, then call a generic reverse-mode autodiff engine. Every operator ships with a hand-coded backward. Your job is to write the forward pass; the framework gives you the gradient. The first time you understand this, deep learning starts to feel like an extremely permissive form of programming.

§4SGD, momentum, Adam, AdamW

Given the gradient, the simplest update is stochastic gradient descent: shuffle the data, draw a minibatch, compute the gradient on that batch, take a step:

$$ \boldsymbol{\theta}_{t+1} \;=\; \boldsymbol{\theta}_t - \eta\, \nabla_{\boldsymbol{\theta}} \mathcal{L}_{\text{batch}}(\boldsymbol{\theta}_t). $$

Stochastic because we use a noisy gradient estimate; the noise is a feature (helps escape narrow minima), not a bug. The learning rate $\eta$ is the single most important hyperparameter. The minibatch size trades off gradient quality for parallelism — modern training uses very large batches (thousands of examples) to saturate GPU throughput.

Momentum (Polyak 1964) maintains a velocity that averages past gradients:

$$ \mathbf{v}_{t+1} = \mu\, \mathbf{v}_t + \nabla \mathcal{L}, \qquad \boldsymbol{\theta}_{t+1} = \boldsymbol{\theta}_t - \eta\, \mathbf{v}_{t+1}, $$

Where (both updates above)

$\boldsymbol{\theta}_t$
The parameter vector at step $t$; $\boldsymbol\theta_{t+1}$ is its value after one update.
$t$
A scalar (integer) — the iteration counter, one per minibatch.
$\eta$
A scalar — the learning rate, the size of each downhill step. Too big diverges, too small crawls.
$\nabla_{\boldsymbol\theta}\mathcal{L}_{\text{batch}}$
A vector the same shape as $\boldsymbol\theta$ — the gradient of the loss on the current minibatch. Points uphill, so we subtract it.
$\mathbf{v}_t$
A vector — the "velocity," an exponentially-decaying running average of past gradients.
$\mu$
A scalar in $[0,1)$ — the momentum coefficient (typically $0.9$). How much of the previous velocity carries over.

which accelerates progress along directions of consistent gradient sign. Nesterov's variant offers a sharper convergence bound on convex problems.

Adam (Kingma & Ba 2014) tracks running estimates of the first and second moments of the gradient and uses them to give each parameter its own adaptive learning rate:

$$ \mathbf{m}_{t+1} = \beta_1 \mathbf{m}_t + (1 - \beta_1) \mathbf{g}_t, \qquad \mathbf{v}_{t+1} = \beta_2 \mathbf{v}_t + (1 - \beta_2)\, \mathbf{g}_t^2, $$
$$ \hat{\mathbf{m}} = \mathbf{m}_{t+1} / (1 - \beta_1^{t+1}), \quad \hat{\mathbf{v}} = \mathbf{v}_{t+1} / (1 - \beta_2^{t+1}), \quad \boldsymbol{\theta}_{t+1} = \boldsymbol{\theta}_t - \eta \frac{\hat{\mathbf{m}}}{\sqrt{\hat{\mathbf{v}}} + \eps}. $$

Where (both Adam lines above)

$\mathbf{g}_t$
A vector — the current gradient (shorthand for $\nabla_{\boldsymbol\theta}\mathcal{L}$); $\mathbf{g}_t^2$ squares it entry by entry.
$\mathbf{m}_{t+1}$
A vector — the running average of the gradient (the "first moment"). Like momentum.
$\mathbf{v}_{t+1}$
A vector — the running average of the squared gradient (the "second moment"). Estimates how large each coordinate's swings are. (Unrelated to the velocity $\mathbf{v}$ in momentum above — same letter, different role.)
$\beta_1, \beta_2$
Scalars in $[0,1)$ — the decay rates for the two averages (defaults $0.9$ and $0.999$). Closer to $1$ = longer memory.
$\hat{\mathbf{m}}, \hat{\mathbf{v}}$
The same averages after "bias correction" — dividing by $1-\beta^{t+1}$ undoes the fact that the averages start at zero and are too small early on.
$\sqrt{\hat{\mathbf{v}}}$
The entrywise square root: each parameter gets divided by its own typical gradient size, giving it an adaptive step.
$\eps$
A tiny scalar (e.g. $10^{-8}$) added to avoid dividing by zero.
$\eta$
A scalar — the global learning rate, as before.

Default $(\beta_1, \beta_2, \eps) = (0.9, 0.999, 10^{-8})$ works for almost everything. AdamW (Loshchilov & Hutter 2019) is Adam with weight decay applied directly to the parameters rather than folded into the gradient — a small but consistent improvement, and the default for transformer training today.

Learning-rate schedules matter as much as the optimizer choice. The 2024 standard for large-model training is a brief linear warmup (a few hundred to a few thousand steps) followed by cosine decay to some small fraction of the peak LR over the rest of training. Some recent work uses warmup-stable-decay (WSD) schedules to allow checkpoint reuse.

§5Initialization and normalization

Two operational details that look like minor engineering but determine whether a deep network trains at all.

Initialization. Weights drawn from too small a distribution make activations vanish through depth; too large, and they explode. The 2010 fix (Glorot & Bengio) chooses the initial weight variance so that the variance of activations and gradients is approximately preserved layer-to-layer. For ReLU networks the corrected variance is $\mathrm{Var}(W_{ij}) = 2/d_{\text{in}}$ (He et al. 2015; Kaiming init). For modern transformers initial weight std is typically $\sim 0.02$, often scaled down further at the residual branches.

Normalization layers rescale activations so the network's internal distributions stay well-conditioned during training.

  • BatchNorm (Ioffe & Szegedy 2015) normalizes each feature across the batch dimension. Dominant in CNNs; awkward for sequence models because the batch statistics couple examples.
  • LayerNorm (Ba et al. 2016) normalizes each example across the feature dimension. Dominant in transformers because it's per-example and works at batch size 1.
  • RMSNorm (Zhang & Sennrich 2019) is LayerNorm without the mean-subtraction step. Slightly cheaper and the modern default in LLaMA-style transformers.

Normalization stabilizes training, allows higher learning rates, and acts as a mild regularizer. It also subtly changes what the optimizer is optimizing — there is a small literature on the interaction between normalization and weight decay that explains why the same recipe needs different hyperparameters with and without it.

§6Why depth, and why this works

The empirical lesson of the last decade: depth buys composition. A shallow network with one wide hidden layer can in principle represent any function (universal approximation) but in practice cannot learn the functions that interesting data actually exhibits. Deeper networks learn hierarchical features — low-level edges combine into shapes, shapes into objects, objects into scenes — and they do so without anyone telling them to.

The other reason deep learning works is more mysterious. Modern networks are typically over-parameterized — they have far more parameters than training examples — and so the loss surface has uncountably many global minima. Gradient descent finds particular ones, biased by initialization and the trajectory of the optimizer, that happen to generalize. The implicit regularization of the optimizer is the part nobody fully understands, and the part that double descent (Chapter I) gestures at. There is still no clean theory.

What we do know is what works engineering-wise. Pick a sensible architecture for the modality; use Kaiming or scaled init; pre-norm residual blocks; AdamW; warmup + cosine; clip the gradient norm at $\sim 1$; train as long as your compute budget allows. The hyperparameters that matter — learning rate, batch size, weight decay — are surprisingly stable across model sizes once you've gotten in the right neighborhood. Most production training recipes you read about have converged on a small set of choices the field discovered between 2015 and 2022.
