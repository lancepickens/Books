Chapter V

Sequences, Attention, and the Transformer

A single architecture, conceptually six lines of code, currently powers every modern language model, most modern image and audio models, and the entire research agenda of the field. It is worth understanding the transformer from the inside.

Before we start

Many problems take a sequence as input — words, audio frames, video frames. The old approach was the recurrent neural network (RNN), which walks through the sequence one position at a time, carrying a hidden state. RNNs were slow (sequential by construction) and hard to train past a few dozen steps. The 2017 transformer architecture replaced them with attention: a mechanism that lets every position in the sequence look at every other position in parallel.

The transformer is now the architecture behind essentially every modern large language model, most image models (Chapter IV), and large parts of audio and video. The chapter builds it from scratch: attention, then multi-head attention, then the transformer block (attention + a small MLP, both with residual connections), then a stack of blocks. It also covers positional encoding (how an inherently permutation-blind operation gets a sense of order back) and introduces the "residual stream" mental model that has become standard for thinking about what transformers do inside.

Terms you'll meet

Sequence
An ordered list of items: words, audio frames, image patches. The general input type of this chapter.
Token
One item in the sequence. For language: a sub-word chunk; for an image: a patch.
Embedding
A learned vector that represents a token. Every token in the vocabulary gets one; the network reads from this table at the start.
RNN (recurrent neural network)
The pre-transformer way to process sequences: keep a hidden state, update it one step at a time. Sequential, hard to train past a few dozen steps.
LSTM, GRU
Gated variants of RNNs (1997 and 2014) that mostly fixed the training problem by routing information through learned gates. Ran the field for two decades.
Hidden state
The running summary an RNN carries from one time step to the next.
Attention
The core operation: each query vector retrieves a weighted combination of value vectors, with weights determined by how similar the query is to each key vector.
Query, key, value (Q, K, V)
Three vectors derived from the input. The query asks "what do I want?"; the key advertises "what do I have?"; the value is the actual content returned.
Self-attention
Attention where Q, K, V are all derived from the same input sequence — every token can look at every other token.
Multi-head attention
Run $H$ independent attention operations in parallel with different learned projections, then concatenate. Each head can specialize on a different pattern.
Causal mask
A constraint that prevents each position from attending to future positions. Needed for next-token prediction; the only architectural difference between encoder and decoder attention.
Softmax (in attention)
Turns raw similarity scores into a probability distribution over keys, so each query gets a normalized weighting.
Transformer block
One layer of the transformer: an attention sublayer plus a small feed-forward sublayer, both wrapped in residual connections and normalization.
Residual stream
The "wide vector" carried at each position through the stack — every layer adds to it, never replaces it. The clean mental model for what a transformer does.
MLP / feed-forward block
The second sublayer of a transformer block: a position-wise two-layer network, applied independently to each token.
SwiGLU
A specific feed-forward block that gates one linear projection by another. Replaced plain GELU MLPs in most modern LLMs.
GELU
A smooth nonlinearity used in transformer feed-forward layers.
LayerNorm
Per-token normalization. Stabilizes training; the default in transformers.
Pre-norm vs. post-norm
Where the normalization goes relative to the residual connection. Pre-norm (inside the residual) trains stably past hundreds of layers; post-norm (the original 2017 recipe) does not.
Positional encoding
Information added to token embeddings so the model knows the order of the sequence (since attention by itself is order-blind).
Sinusoidal embedding
The original positional encoding: a fixed pattern of sines and cosines at different frequencies, added to each token embedding.
RoPE (Rotary Position Embedding)
The modern positional encoding used in Llama-family models. Rotates query and key vectors by an angle proportional to their position, so attention scores depend only on relative offset.
Decoder-only, encoder-only, encoder–decoder
Three configurations of the same building block. Decoder-only with a causal mask is the GPT / Llama / DeepSeek family; encoder-only with bidirectional attention is BERT; encoder–decoder is T5 and the original translation transformer.
BERT, GPT
The two early breakout transformer model families: BERT (2018, encoder-only, masked-word pretraining) and GPT (2018-onward, decoder-only, next-token pretraining).
FlashAttention
A clever 2022 reimplementation of exact attention that's much more memory-efficient. Lets training and inference run on longer sequences.
KV cache
At inference time, the cached keys and values from earlier tokens so we don't recompute them. The reason serving an LLM is fast after the first token.
State-space model / Mamba
A 2023 alternative architecture: a modernized RNN with a parallel-scan training algorithm. Linear in sequence length where attention is quadratic. Competitive at small scales, jury still out at frontier.
Induction head
A specific attention pattern, identified by interpretability researchers, that implements simple "look back and complete the pattern" behavior. The poster child for mechanistic interpretability (revisited in Chapter X).

§1Sequences, and why RNNs lost

Many problems take a sequence as input — language, audio, time series, video. The natural early approach was the recurrent neural network: maintain a hidden state $\mathbf{h}_t$ that updates as you walk through the sequence,

$$ \mathbf{h}_t \;=\; \sigma\bigl(\mathbf{W}_h \mathbf{h}_{t-1} + \mathbf{W}_x \mathbf{x}_t + \mathbf{b}\bigr). $$

This is a beautiful idea — a model with state that can in principle remember arbitrarily long histories. The execution had two flaws.

First, training: gradients through $T$ time steps multiply $T$ factors of $\mathbf{W}_h$, which either vanish (most directions in $\mathbf{W}_h$) or explode (a few directions). The LSTM (Hochreiter & Schmidhuber 1997) and GRU (Cho et al. 2014) routed information through gates designed to keep gradients alive, and ran the field for two decades.

Second, parallelism: even with gates, recurrence is sequential. You cannot compute $\mathbf{h}_{100}$ until you've computed $\mathbf{h}_{99}$. On modern hardware, where throughput comes from massive parallel matrix multiplies, this is a structural disaster.

The transformer (Vaswani et al. 2017) discarded recurrence and gates entirely. Replace the sequential dependence with attention, a parallelizable mechanism that lets any position look directly at any other in one layer. The result trained dramatically faster and learned better. Within five years, attention-based models had displaced RNNs essentially everywhere.

§2Attention as a soft lookup

Think of a dictionary. Given a query string, you return the value paired with the most similar key. Attention is the differentiable version: queries and keys are vectors; the answer is a weighted average of values, weighted by softmax-normalized similarity between query and each key.

Formally, given queries $\mathbf{Q} \in \mathbb{R}^{n \times d_k}$, keys $\mathbf{K} \in \mathbb{R}^{m \times d_k}$, and values $\mathbf{V} \in \mathbb{R}^{m \times d_v}$:

$$ \mathrm{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) \;=\; \mathrm{softmax}\!\Bigl(\tfrac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d_k}}\Bigr)\, \mathbf{V}. $$

Where

$\mathbf{Q}\mathbf{K}^\top \in \mathbb{R}^{n \times m}$
Pairwise similarity scores: entry $(i, j)$ measures how much query $i$ looks like key $j$.
$1/\sqrt{d_k}$
Scaling: without it, dot products grow as $\sqrt{d_k}$ in magnitude, pushing softmax into saturated regions where gradients vanish.
softmax (row-wise)
Converts each row to a probability distribution over keys. Each query winds up with a soft selection over values.
output
$n \times d_v$. Row $i$ is the value retrieved by query $i$.

The naming comes from the dictionary metaphor — query, key, value — and it is more than metaphor: any soft retrieval problem (look up by image content, search by sentence) has this shape.

§3Self-attention

In self-attention, all of $\mathbf{Q}, \mathbf{K}, \mathbf{V}$ come from the same input sequence. Given token embeddings $\mathbf{X} \in \mathbb{R}^{n \times d}$, learned projection matrices $\mathbf{W}_Q, \mathbf{W}_K, \mathbf{W}_V \in \mathbb{R}^{d \times d_k}$ produce

$$ \mathbf{Q} = \mathbf{X}\mathbf{W}_Q, \qquad \mathbf{K} = \mathbf{X}\mathbf{W}_K, \qquad \mathbf{V} = \mathbf{X}\mathbf{W}_V, $$

and the attention layer's output is $\mathrm{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V})$. Each token's new representation is a weighted average of every other token's value, weighted by content similarity. Information flows from anywhere to anywhere in a single layer.

For language modeling (predict next token), we must prevent each position from peeking at future tokens. The fix is a causal mask that sets $\mathbf{Q}\mathbf{K}^\top_{ij} = -\infty$ for $j > i$ before the softmax, making the corresponding attention weight zero. This is the only difference between encoder self-attention (BERT-style; bidirectional) and decoder self-attention (GPT-style; causal).

Multi-head attention runs $H$ independent attention operations in parallel with different projection matrices, then concatenates the outputs. Each head can specialize on a different relational pattern (Olsson et al. 2022 — induction heads, the simplest example). Typical modern models have $H = 8$ to $H = 64$ heads.

The cost of self-attention is $O(n^2 d)$ in compute and $O(n^2)$ in memory: the full attention matrix has size $n \times n$. For $n = 4096$ this is fine; for $n = 1{,}000{,}000$ tokens it is the dominant bottleneck of long-context inference. FlashAttention (Dao et al. 2022) reordered the computation to be memory-efficient without changing the math; long-context techniques (sliding window, sparse, KV cache compression) and alternative architectures (state-space models like Mamba; Gu & Dao 2023) attack the same wall.

§4The transformer block

A single transformer block is two sublayers, each wrapped in a residual connection with a layer norm:

$$ \mathbf{x}' \;=\; \mathbf{x} + \mathrm{Attn}\bigl(\mathrm{LN}(\mathbf{x})\bigr), \qquad \mathbf{x}'' \;=\; \mathbf{x}' + \mathrm{MLP}\bigl(\mathrm{LN}(\mathbf{x}')\bigr). $$

The MLP is a position-wise two-layer feed-forward network — typically 4× the embedding dimension wide — applied independently to every token. The modern feed-forward block is SwiGLU (Shazeer 2020):

$$ \mathrm{SwiGLU}(\mathbf{x}) \;=\; \bigl(\mathrm{Swish}(\mathbf{x}\mathbf{W}_1) \odot \mathbf{x}\mathbf{W}_2\bigr)\, \mathbf{W}_3, $$

which gates the activation by another linear projection — empirically a meaningful boost over a plain GELU MLP. Normalization is almost always pre-norm (apply LN inside the residual) for training stability; post-norm (LN after the residual addition) is the original Vaswani recipe and is harder to train past 12 layers without warmup tricks.

The full decoder-only transformer is just $L$ of these blocks stacked, with a token embedding at the start and an output projection ("unembedding") at the end:

$$ \mathbf{x}_0 = \mathrm{Embed}(\text{tokens}) + \mathrm{Pos}, \quad \mathbf{x}_\ell = \mathrm{Block}_\ell(\mathbf{x}_{\ell-1}), \quad \text{logits} = \mathbf{x}_L \mathbf{W}_U. $$

§5Positional encoding: from sinusoids to RoPE

Self-attention is permutation-equivariant: shuffle the input tokens, the output shuffles correspondingly. For sequences, that is the wrong inductive bias — order matters. We inject positional information explicitly.

The original transformer used fixed sinusoidal embeddings at every position, added to the token embedding:

$$ \mathrm{PE}(p, 2k) = \sin(p / 10000^{2k/d}), \qquad \mathrm{PE}(p, 2k+1) = \cos(p / 10000^{2k/d}). $$

Later models used learned absolute position embeddings, then relative position bias (T5; ALiBi). The modern dominant choice in LLaMA-family models is RoPE (Rotary Position Embedding; Su et al. 2021): rotate the query and key vectors by an angle proportional to position, encoding position into the angle of $\mathbf{Q}$ and $\mathbf{K}$ rather than adding to the embedding. The attention score between positions $i$ and $j$ then depends only on $i - j$, the relative offset — an elegant geometric property that extrapolates better to longer sequences than absolute encoding.

§6The residual stream view

The Anthropic interpretability program (Elhage et al. 2021) reframed the transformer in a way that has become standard among mechanistic-interpretability researchers and is the cleanest mental model for what transformers actually do.

The hidden state $\mathbf{x}_\ell$ is a high-dimensional residual stream: a wide vector at each position that every layer reads from and writes to. Attention heads move information between positions by reading at one position and writing at another. MLPs process information at a single position, transforming whatever was written into the stream. The residual stream itself is just the running sum of all the writes.

This is more than a metaphor. It explains why dropping in a "skip connection" identity matters so much (information persists), why early-layer interventions can have global effects (every later layer reads them), and why so-called circuits — small subnetworks implementing identifiable behaviors like indirect-object identification or pattern-matching induction — are findable. We will return to circuits and mechanistic interpretability in Chapter X.

§7Three flavors, one architecture

  • Encoder-only (BERT-family). Bidirectional self-attention; trained to fill in masked tokens. Used for classification, embeddings, retrieval (Chapter IX).
  • Decoder-only (GPT-family, the modern default). Causal self-attention; trained to predict the next token. All modern LLMs — GPT-4 / Claude / Llama / DeepSeek / Gemini — fit here.
  • Encoder–decoder (T5, original "Attention Is All You Need" architecture). Encoder reads the input bidirectionally, decoder generates output with cross-attention to the encoder. Currently dominant for translation and structured generation; less popular for general-purpose chatbots, though still used in some image–text models.

The pattern of "one architecture, three configurations" is unusual in machine learning. Most prior eras had different architectures for different problems. The transformer's success across modalities is part of why people use the word foundation model: one trainable thing that, given the right data, becomes a vision system, a code model, a translator, a chat assistant. Chapter VI is about what training one looks like.
