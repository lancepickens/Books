Chapter I

Foundations: Data, Loss, and the Learning Problem

Every machine learning system is the same three pieces — a dataset, a family of functions, a loss to minimize. Everything else is the artistry of choosing them well.

Before we start

"Machine learning" means building a model that gets reliably better at predictions on examples it has not seen. Memorizing the training examples is easy; the science is the part where the model also does well on new ones. This chapter introduces the frame the whole book uses — every method we'll meet is some choice of data, a family of candidate functions, and a loss that measures how wrong each prediction is. An algorithm searches the family for the function that makes the loss small.

The rest of the chapter explains the central worry — why a model that fits the training data perfectly can still fail on new data — and the discipline (separating training, validation, and test sets) that keeps us honest about it.

Terms you'll meet

Model
A function that takes an input and produces a prediction. Has tunable knobs (parameters).
Feature / input
The information the model gets. For an image, the pixel values; for text, the tokens; for a customer record, the columns.
Label / target
The answer we want the model to produce: the class, the price, the next word.
Training / validation / test set
Three disjoint slices of the data. The model trains on the first; we tune choices using the second; we report performance on the third (touched once).
Loss function
A number that says how wrong a single prediction is. Smaller is better. Squared error and cross-entropy are the two you'll see most.
Risk / population risk
The average loss the model would have on fresh data from the same source. What we actually care about — but cannot compute, since we never see all possible data.
Empirical risk
The average loss on the training examples we do have. What we can compute and minimize.
ERM (Empirical Risk Minimization)
The recipe: pick the parameters that make the empirical risk smallest, and hope the population risk follows.
Generalization
The model's tendency to do well on data it hasn't seen — the gap between empirical risk and population risk.
Overfitting
The model learns the training set's quirks instead of the underlying pattern. Training error small, test error large.
Underfitting
The model is too simple to capture the pattern. Training and test error both large.
Bias (of a model)
How wrong the model is on average across many training sets. High bias = too rigid.
Variance (of a model)
How much the trained model wobbles when you reshuffle the training data. High variance = too sensitive.
Double descent
A 2019 observation: for very large neural nets, test error can drop again past the point where the model fits training perfectly — breaking the textbook U-shaped curve.
Cross-validation
Rotate which slice of data acts as validation, average the result. Useful when data is scarce.
Leakage
When information from validation or test data sneaks into training, inflating apparent performance. The silent killer.
Supervised learning
Labeled examples: you have inputs and the right answers.
Unsupervised learning
Inputs only, no labels. The job is to find structure (clusters, low-dimensional summaries).
Self-supervised learning
Invent a "label" from the input itself (predict the next word, fill in a missing patch). The trick behind every modern foundation model.
Reinforcement learning
An agent takes actions in an environment and gets a numerical reward. Learn a policy that picks rewarding actions.
Foundation model
A large model pretrained on broad data that can be reused (with light fine-tuning or prompting) for many downstream tasks.
Token
The atomic unit a language model reads or writes — usually a sub-word chunk, not a whole word. Chapter VI covers tokenization properly.

§1What "learning" means

A useful operating definition: a model learns if, after exposure to examples, it makes better predictions on examples it has not seen. The phrase has not seen is the entire game. Memorization is easy; generalization is the science.

Formalize this. There is a fixed but unknown distribution $\mathcal{D}$ over input–output pairs $(\mathbf{x}, y) \in \mathcal{X} \times \mathcal{Y}$. We draw a training set

$$ \mathcal{S} \;=\; \{(\mathbf{x}_i, y_i)\}_{i=1}^{N} \stackrel{\text{i.i.d.}}{\sim} \mathcal{D}. $$

We pick a function $f_\theta: \mathcal{X} \to \mathcal{Y}$ from some family parameterized by $\boldsymbol{\theta}$, and a loss $\ell(\hat y, y)$ that says how bad it is to predict $\hat y$ when the truth is $y$. The thing we actually care about — what the model would lose on a fresh draw from the distribution — is the population risk or true risk:

$$ R(\boldsymbol{\theta}) \;=\; \mathbb{E}_{(\mathbf{x}, y) \sim \mathcal{D}} \bigl[\, \ell(f_\theta(\mathbf{x}), y) \,\bigr]. $$

We cannot evaluate this; we don't know $\mathcal{D}$. What we can compute is the average loss on the training sample:

$$ \hat R(\boldsymbol{\theta}) \;=\; \frac{1}{N} \sum_{i=1}^{N} \ell(f_\theta(\mathbf{x}_i), y_i). $$

Where

$\mathcal{X}, \mathcal{Y}$
Input and output spaces. For pixels, $\mathcal{X} = \mathbb{R}^{H\times W\times 3}$; for text, $\mathcal{X}$ is a sequence of tokens; for a classifier, $\mathcal{Y} = \{1, \dots, K\}$.
$\boldsymbol{\theta}$
Parameters of the model. Could be ten coefficients (linear regression) or a trillion weights (a frontier LLM).
$\ell$
Per-example loss. Squared error $(\hat y - y)^2$ for regression; cross-entropy $-\log p_\theta(y \mid \mathbf{x})$ for classification.
$R$ vs $\hat R$
$R$ is what we want; $\hat R$ is what we have. The gap between them is the central object of learning theory.

The recipe called empirical risk minimization (ERM) is exactly what it sounds like: pick $\boldsymbol{\theta}$ that minimizes $\hat R$. Then hope $R(\boldsymbol{\theta})$ is also small. The book is mostly about why the gap between $R$ and $\hat R$ stays manageable — and the few cases where it doesn't.

§2The triple: data, function class, loss

Every supervised method we will discuss is some choice of three things.

  • Data — the pairs $(\mathbf{x}_i, y_i)$. For deep learning, more is almost always better. The 2010s lesson is that the data distribution shapes everything; the 2020s lesson is that data quality, mixed correctly, can matter more than raw volume (Chapter VI).
  • Function class $\mathcal{H} = \{f_\theta : \boldsymbol{\theta} \in \Theta\}$ — the set of candidate hypotheses. Linear functions; trees; neural networks of a fixed architecture. Choosing $\mathcal{H}$ is choosing a set of inductive biases: assumptions about what kind of functions are likely correct. The transformer's success is largely a story about its inductive biases turning out to be the right ones for language and beyond.
  • Loss $\ell$ — what counts as a good prediction. The choice is partly engineering convenience (we like smooth, convex losses we can differentiate) and partly Bayesian: most losses are negative log-likelihoods under some probabilistic model, so picking a loss is picking a noise model.

An optimization algorithm — Chapter III — searches $\mathcal{H}$ for the $\boldsymbol{\theta}$ that minimizes $\hat R$. For most modern models, that algorithm is some flavor of stochastic gradient descent.

§3Losses are probability models in disguise

Worth a moment because it explains why two losses — squared error and cross-entropy — show up over and over.

For regression: suppose $y = f^*(\mathbf{x}) + \eps$ with $\eps \sim \mathcal{N}(0, \sigma^2)$. The negative log-likelihood of an observation is

$$ -\log p(y \mid \mathbf{x}, \boldsymbol{\theta}) \;=\; \frac{1}{2\sigma^2}\bigl(y - f_\theta(\mathbf{x})\bigr)^2 + \text{const}. $$

Minimizing squared error is maximum-likelihood under Gaussian noise. The variance $\sigma^2$ drops out of the optimization — it sets the units of the loss, not its shape.

For classification: a model that emits a probability $p_\theta(y \mid \mathbf{x})$ over $K$ classes has per-example negative log-likelihood

$$ \ell(\mathbf{x}, y) \;=\; -\log p_\theta(y \mid \mathbf{x}). $$

This is the cross-entropy loss. Aggregated over $N$ examples, it is the loss every modern language model is trained on, and the loss inside every classification head you will write.

The probabilistic view is more than a curiosity: it tells you what your model is implicitly assuming about the world's noise, and what to do when the assumption fails (Tukey-style robust losses for heavy-tailed regression; class-balanced cross-entropy for imbalanced classification).

§4Generalization and the bias–variance picture

Why doesn't training error tell you everything you want to know? Because a sufficiently flexible model can drive $\hat R$ to zero by memorizing the sample, while $R$ stays high. The classical story explains why with a decomposition.

For squared error and a regression target $y = f^*(\mathbf{x}) + \eps$ with noise variance $\sigma^2$, the expected loss at a fixed test point $\mathbf{x}_0$ — averaged over training sets — decomposes exactly:

$$ \mathbb{E}_{\mathcal{S}}\bigl[\, (f_{\hat\theta}(\mathbf{x}_0) - y_0)^2 \,\bigr] \;=\; \underbrace{\bigl(\mathbb{E}_{\mathcal{S}}[f_{\hat\theta}(\mathbf{x}_0)] - f^*(\mathbf{x}_0)\bigr)^2}_{\text{bias}^2} \;+\; \underbrace{\mathrm{Var}_{\mathcal{S}}\bigl[f_{\hat\theta}(\mathbf{x}_0)\bigr]}_{\text{variance}} \;+\; \underbrace{\sigma^2}_{\text{irreducible}}. $$

Where

bias
How wrong the average trained model is. Models too simple to express $f^*$ have high bias regardless of how much data they see.
variance
How much the trained model wobbles as you reshuffle the training set. Models with many parameters and little data have high variance.
$\sigma^2$
Noise in the data itself. The best possible model still has expected loss $\sigma^2$ at every point. Achievable error has a floor.

The classical regime says: small, flexible models have high bias and low variance; large, flexible models have low bias and high variance; the test-error curve is U-shaped in model size. The sweet spot lives near the bottom of the U.

Double descent — the modern wrinkle

For deep networks, test error sometimes decreases again past the point where the model fits training data perfectly. The single U becomes a U followed by a long descent (Belkin et al. 2019; Nakkiran et al. 2021). The reason — heavily simplified — is that among the infinitely many parameter settings that fit training perfectly, the optimizer's implicit bias selects ones with small norm, which generalize. This effect underlies why over-parameterized networks work at all; classical bias–variance is the right intuition for small models, double descent for big ones.

§5Train, validation, test, and the dangers of leakage

You cannot estimate generalization from data the model has trained on. The standard partition:

  • Training set. What the optimizer sees. Used to compute gradients of $\hat R$.
  • Validation set. What you tune hyperparameters and architecture on. You evaluate many candidates against it; if you tune too aggressively, the validation set leaks information into your model and stops being an honest estimate of $R$.
  • Test set. Touched once, at the end. Your reported error.

For small datasets, $k$-fold cross-validation rotates the validation split to give a tighter error estimate. For modern foundation-model work, the test set is increasingly an evaluation suite (Chapter VIII): a battery of held-out benchmarks the model is judged on rather than a clean held-out split, because frontier models are trained on essentially everything publicly readable.

Leakage is the silent killer. Examples:

  • Features computed using future information your deployed model would not have.
  • Duplicates between train and test (a chronic problem in web-scraped data; see the C4-near-duplicates literature for how bad it gets).
  • The train/test split made after some preprocessing was fit on all data — scalers, vocabularies, even feature selection.
  • For time series: random splits where the model effectively sees the future during training.

The discipline is: every transformation downstream of the split must be fit on training data only.

§6What problem are you actually solving?

Three classical regimes:

  • Supervised learning — labeled pairs $(\mathbf{x}_i, y_i)$. Most of this book.
  • Unsupervised learning — unlabeled $\mathbf{x}_i$. Clustering, density estimation, dimensionality reduction. Important historically, smaller share of modern work.
  • Reinforcement learning — an agent acts in an environment and learns from a scalar reward. Chapters VII and VIII pull RL into the LLM stack via RLHF and reasoning models.

The 2020s frame puts self-supervised learning at the center: invent a supervised problem from unlabeled data (the next token; the masked patch; whether two augmented views came from the same image) and use the resulting representations for whatever you actually wanted to do. Every modern foundation model is pretrained this way. Chapter VI takes this seriously.

§7A preview of the ladder

The next nine chapters climb from the simplest case to the contemporary stack.

  1. Linear models (Ch. II): when $f_\theta$ is linear in $\boldsymbol{\theta}$, everything has a closed form and the geometry is clean. Kernels stretch this far further than it has any right to go.
  2. Neural networks (Ch. III): compose differentiable maps and fit by gradient descent. Backprop makes it tractable; the optimizer is what makes it train.
  3. Vision (Ch. IV): convolutions for translation-equivariant pixels, then ViTs that treat images as sequences of patches.
  4. Transformers (Ch. V): self-attention as a soft lookup, residual streams, decoder-only blocks — the single architecture behind almost everything in the modern stack.
  5. Pretraining at scale (Ch. VI): self-supervised objectives, tokenizers, scaling laws, the alternatives.
  6. Alignment (Ch. VII): turn a next-token predictor into a useful assistant via SFT, RLHF, DPO.
  7. Reasoning (Ch. VIII): the 2024 shift to training models that spend test-time compute thinking before answering.
  8. Agents and retrieval (Ch. IX): tool use, planning loops, RAG — the stack that turns chat models into systems.
  9. Generative and the frontier (Ch. X): diffusion, multimodal, video, mechanistic interpretability, what nobody has figured out yet.

The frame survives the entire climb. Every method below chooses some data, some function class, some loss, and minimizes the loss on the data. What changes is what we are willing to call data, how rich a function class we can afford to fit, and how cleverly we have defined "loss" to encode what we actually want.

Fit polynomials of varying degree to a noisy sample drawn from a fixed true function $f^*(x) = \sin(2\pi x)$ with Gaussian noise. Low degree: underfit, high training and test error (high bias). High degree: overfits the specific noise sample, low training error, high test error (high variance). Resample several times to watch the high-degree fit thrash while the low-degree fit stays stable — that is variance, visualized.
