Chapter II

Linear Models, Regularization, and the Kernel Bridge

The first nontrivial class of models — and the last one we can solve in closed form. Everything past this chapter will be solved by gradient descent. Most of what makes that work was visible already here.

Before we start

This chapter is about the simplest workable machine-learning models. Linear regression predicts a number as a weighted sum of inputs; logistic regression does the same for two-class classification, with a squashing function so the output is a probability. Both have closed-form (or near-closed-form) solutions — no iterative training needed. Adding a regularization term that penalizes large weights stops them overfitting.

The second half introduces the kernel trick: a way to make linear models behave nonlinearly, by computing similarities between examples instead of building explicit nonlinear features. Kernels ran the field from 1995 to about 2010 before deep nets took over; their geometric ideas (inner products, projections, balls and corners) come back in every later chapter.

Terms you'll meet

Linear regression
Predict a continuous number as a weighted sum of input features.
Logistic regression
Same weighted sum, but pushed through a sigmoid to give a probability between 0 and 1. Used for binary classification.
Weight / coefficient
The number the model multiplies each input by. Collectively, the parameters.
Sigmoid
The S-shaped function $\sigma(z) = 1/(1+e^{-z})$ that smoothly maps any real number to the interval (0, 1).
Softmax
The multi-class generalization of sigmoid: turn $K$ real numbers into $K$ probabilities that sum to 1.
Cross-entropy
The standard loss for classification: the negative log of the probability the model assigned to the true class. Used in essentially every classifier you'll see.
Normal equations
The linear system $\mathbf{X}^\top\mathbf{X}\,\mathbf{w} = \mathbf{X}^\top\mathbf{y}$ whose solution gives the best linear-regression weights. One matrix solve, no iteration.
OLS (ordinary least squares)
Plain linear regression with squared-error loss and no regularizer.
Pseudoinverse
A generalized inverse $\mathbf{X}^+$ that works even when $\mathbf{X}$ isn't square or invertible. Lets the normal equations give a unique "minimum-norm" answer.
Regularization
Adding a penalty on the size of the weights to the loss. Keeps the model from overfitting and tames the geometry of the optimum.
Ridge / L2 / Tikhonov
Penalize the sum of squared weights. Shrinks all weights smoothly toward zero.
Lasso / L1
Penalize the sum of absolute weights. Drives some weights to exactly zero — useful when you want a sparse model that picks a few features.
Elastic net
A weighted mix of ridge and lasso. Useful when features are correlated.
Shrinkage / sparsity
Shrinkage: pulling weights toward zero. Sparsity: many weights are exactly zero.
Feature map $\phi$
A transformation that turns raw inputs into a (possibly much higher-dimensional) feature vector. Polynomial expansions are the classic example.
Kernel
A function $k(\mathbf{x}, \mathbf{x}')$ that secretly computes an inner product in some feature space, without ever building the feature vectors explicitly.
Kernel trick
The observation that many algorithms only ever need inner products between examples — so any kernel will do, including ones whose implicit feature space is infinite-dimensional.
Gram matrix $\mathbf{K}$
The $N \times N$ table of kernel values between every pair of training examples.
Representer theorem
The mathematical fact that says the best kernelized solution can always be written as a weighted sum over the training examples — so you never need to work with the implicit features themselves.
RBF / Gaussian kernel
The kernel $k(\mathbf{x}, \mathbf{x}') = \exp(-\norm{\mathbf{x} - \mathbf{x}'}^2 / (2\ell^2))$ — a smooth bump at every training point. Default-go-to kernel.
Length scale $\ell$
The dominant tuning knob of an RBF kernel: how far away two points have to be before the model treats them as unrelated.
SVM (support vector machine)
A classifier that finds the maximum-margin separating boundary. Only the closest examples — the "support vectors" — matter for the final prediction.
Gaussian process
A kernel method for regression that returns a calibrated uncertainty along with each prediction. Still the right tool for small-data regression.
Neural Tangent Kernel
A kernel that approximates what an infinite-width neural net does during training. Connects deep nets back to the kernel theory of this chapter.

§1Linear regression as projection

The setup. Stack training inputs into a matrix $\mathbf{X} \in \mathbb{R}^{N \times d}$, targets into a vector $\mathbf{y} \in \mathbb{R}^N$. The linear model predicts $\hat{\mathbf{y}} = \mathbf{X}\mathbf{w}$ for some weight vector $\mathbf{w} \in \mathbb{R}^d$. The squared-error loss is

$$ \mathcal{L}(\mathbf{w}) \;=\; \tfrac{1}{2}\, \norm{\mathbf{y} - \mathbf{X}\mathbf{w}}^2. $$

Setting the gradient $\nabla_{\mathbf{w}} \mathcal{L} = -\mathbf{X}^\top(\mathbf{y} - \mathbf{X}\mathbf{w})$ to zero gives the normal equations:

$$ \mathbf{X}^\top \mathbf{X}\, \mathbf{w} \;=\; \mathbf{X}^\top \mathbf{y}, \qquad \hat{\mathbf{w}} \;=\; (\mathbf{X}^\top \mathbf{X})^{-1} \mathbf{X}^\top \mathbf{y}. $$

Where

$\mathbf{X}$
An $N \times d$ matrix — one row per training example, one column per input feature. Called the design matrix.
$\mathbf{y}$
A vector in $\mathbb{R}^N$ — the target value for each of the $N$ examples, stacked.
$\mathbf{w}$
A vector in $\mathbb{R}^d$ — one weight per feature. The thing we solve for.
$\hat{\mathbf{w}}$
A vector in $\mathbb{R}^d$ — the best-fit weights (the hat means "estimated from data").
$\mathbf{X}^\top$
The transpose of $\mathbf{X}$ (rows become columns): a $d \times N$ matrix.
$\mathbf{X}^\top\mathbf{X}$
A $d \times d$ matrix — the feature-by-feature covariance pattern. Inverting it solves the system.
$\nabla_{\mathbf{w}}\mathcal{L}$
The gradient: a vector in $\mathbb{R}^d$ holding the partial derivative of the loss with respect to each weight. Setting it to zero locates the minimum.

Geometrically: the fitted prediction $\hat{\mathbf{y}} = \mathbf{X}\hat{\mathbf{w}}$ is the orthogonal projection of $\mathbf{y}$ onto the column space of $\mathbf{X}$. The residual $\mathbf{y} - \hat{\mathbf{y}}$ is orthogonal (at a right angle) to every column of $\mathbf{X}$ — that is exactly the normal equation. The matrix $\mathbf{H} = \mathbf{X}(\mathbf{X}^\top\mathbf{X})^{-1}\mathbf{X}^\top$ is the hat matrix, the projector onto column space. Linear regression is one matrix solve. There are no hyperparameters in the unregularized case.

When $\mathbf{X}^\top\mathbf{X}$ is singular

If $d > N$ (more features than examples), or if columns of $\mathbf{X}$ are linearly dependent, $\mathbf{X}^\top\mathbf{X}$ is rank-deficient (a.k.a. singular, non-invertible) and the closed form fails. Either drop redundant columns, or — almost always cleaner — regularize. The Moore–Penrose pseudoinverse $\mathbf{X}^+$ gives the minimum-norm solution; ridge regression below gives the small-norm preferred-direction solution.

§2Logistic regression for classification

For binary classification with labels $y \in \{0, 1\}$, the linear model produces a real number $\mathbf{w}^\top\mathbf{x}$ that we squash through the sigmoid to get a probability:

$$ p_\mathbf{w}(y = 1 \mid \mathbf{x}) \;=\; \sigma(\mathbf{w}^\top \mathbf{x}) \;=\; \frac{1}{1 + e^{-\mathbf{w}^\top \mathbf{x}}}. $$

The negative log-likelihood is the binary cross-entropy:

$$ \mathcal{L}(\mathbf{w}) \;=\; -\sum_{i=1}^{N} \bigl[\, y_i \log \sigma(\mathbf{w}^\top \mathbf{x}_i) + (1 - y_i) \log(1 - \sigma(\mathbf{w}^\top \mathbf{x}_i)) \,\bigr]. $$

Where

$\mathbf{w}^\top\mathbf{x}_i$
A scalar — the dot product of the weight vector with example $i$'s features (the "score" before squashing).
$\sigma(\cdot)$
The sigmoid, turning the score into a probability in $(0,1)$.
$y_i$
A scalar that is $0$ or $1$ — the true label of example $i$. Exactly one of the two bracketed terms survives for each example.
$\sum_{i=1}^N$
Sum over all $N$ training examples; the loss is their total.
convex
The loss surface is bowl-shaped — one global minimum, no local traps — so gradient methods are guaranteed to find the best $\mathbf{w}$.

The loss is convex in $\mathbf{w}$, with gradient

$$ \nabla_{\mathbf{w}} \mathcal{L} \;=\; \mathbf{X}^\top (\boldsymbol{\sigma} - \mathbf{y}), $$

where $\boldsymbol{\sigma}$ is the vector of $\sigma(\mathbf{w}^\top \mathbf{x}_i)$. No closed form, but Newton's method converges in a handful of steps (each step is a weighted least-squares problem — iteratively reweighted least squares). For $K$ classes the natural generalization is the softmax:

$$ p_\mathbf{W}(y = k \mid \mathbf{x}) \;=\; \frac{e^{\mathbf{w}_k^\top \mathbf{x}}}{\sum_{j=1}^{K} e^{\mathbf{w}_j^\top \mathbf{x}}}, $$

Where

$K$
A scalar (positive integer) — the number of classes.
$\mathbf{w}_k$
A vector in $\mathbb{R}^d$ — the weight vector for class $k$. Stacked over all classes they form the matrix $\mathbf{W}$.
$e^{\mathbf{w}_k^\top\mathbf{x}}$
The exponential of class $k$'s score — always positive, so the ratios are valid probabilities.
denominator $\sum_j e^{\mathbf{w}_j^\top\mathbf{x}}$
The normalizer summing over all classes, forcing the $K$ outputs to add up to $1$.

with a categorical cross-entropy loss. Logistic-softmax classification is the head sitting on top of almost every modern classifier; everything between input and the final linear layer is feature extraction.

§3Regularization is geometry

Adding a penalty on $\norm{\mathbf{w}}$ to the loss changes the geometry of the optimum. Two penalties dominate.

Ridge regression (Tikhonov, $\ell_2$):

$$ \hat{\mathbf{w}}_{\text{ridge}} \;=\; \argmin_{\mathbf{w}} \;\tfrac{1}{2}\,\norm{\mathbf{y} - \mathbf{X}\mathbf{w}}^2 + \lambda\, \norm{\mathbf{w}}_2^2 \;=\; (\mathbf{X}^\top \mathbf{X} + 2\lambda \mathbf{I})^{-1} \mathbf{X}^\top \mathbf{y}. $$

Lasso (Tibshirani 1996, $\ell_1$):

$$ \hat{\mathbf{w}}_{\text{lasso}} \;=\; \argmin_{\mathbf{w}} \;\tfrac{1}{2}\,\norm{\mathbf{y} - \mathbf{X}\mathbf{w}}^2 + \lambda\, \norm{\mathbf{w}}_1. $$

Where

$\arg\min_{\mathbf{w}}$
"The value of $\mathbf{w}$ that minimizes what follows" — not the minimum value itself, but the weights that achieve it.
$\lambda \geq 0$
A scalar — the regularization strength. $\lambda = 0$ recovers ordinary least squares; $\lambda \to \infty$ shrinks $\mathbf{w}$ to zero.
$\norm{\mathbf{w}}_2^2$
The squared $\ell_2$ norm: the sum of squared weights. Smooth — adds $2\lambda \mathbf{I}$ to the normal equations.
$\norm{\mathbf{w}}_1$
The $\ell_1$ norm: the sum of absolute values. Non-smooth at zero — exactly the property that drives weights to be exactly zero. Lasso fits are sparse.
$\mathbf{I}$
The $d \times d$ identity matrix (ones on the diagonal, zeros elsewhere). Adding $2\lambda\mathbf{I}$ nudges $\mathbf{X}^\top\mathbf{X}$ back to invertibility.

The geometric picture is the standard one. The unregularized loss has elliptical level sets centered on the OLS optimum. The ridge constraint $\norm{\mathbf{w}}_2 \leq C$ is a ball; the constrained optimum is where the smallest level set touches the ball, generically at a point on its smooth spherical surface away from the axes — no zero weights. The lasso constraint $\norm{\mathbf{w}}_1 \leq C$ is a diamond with corners on the axes; the touching point sits on a corner whenever the loss ellipsoid is oriented at all toward the axis, killing that coordinate. Sparsity from $\ell_1$ is a corner phenomenon.

Both penalties shrink coefficients toward zero; ridge does so smoothly, lasso aggressively and selectively. Elastic net (Zou & Hastie 2005) combines both — useful when features are correlated. Regularization more generally is the lever that controls bias and variance: more regularization → more bias, less variance.

§4Beyond linear: the kernel trick

Linear models are weak; their power explodes the moment you allow nonlinear features. The naïve route is to engineer features $\phi(\mathbf{x}) \in \mathbb{R}^D$ — polynomial terms, indicator buckets, interaction products — and fit a linear model in $\phi$. For high-order polynomials in many input dimensions, $D$ grows combinatorially and the approach becomes infeasible.

The kernel trick is the observation that for many problems, predictions and the loss depend on the feature vectors only through inner products $\phi(\mathbf{x})^\top \phi(\mathbf{x}')$ — never through the feature vectors themselves. If we can evaluate that inner product directly, without ever materializing $\phi$, we are free to use feature spaces of enormous (even infinite) dimension.

A kernel $k(\mathbf{x}, \mathbf{x}')$ is exactly such a function: symmetric, positive-semidefinite (so the Gram matrix $\mathbf{K}_{ij} = k(\mathbf{x}_i, \mathbf{x}_j)$ is PSD), and corresponding to some feature map $\phi$ in a Hilbert space. The representer theorem (Kimeldorf & Wahba 1971) says the optimal weight vector for any regularized empirical-risk problem with a kernel lives in the span of the training data:

$$ \mathbf{w}^* \;=\; \sum_{i=1}^{N} \alpha_i\, \phi(\mathbf{x}_i), \qquad f^*(\mathbf{x}) \;=\; \sum_{i=1}^{N} \alpha_i\, k(\mathbf{x}_i, \mathbf{x}). $$

Where

$\phi(\mathbf{x})$
The feature map — turns a raw input into a (possibly infinite-dimensional) feature vector. Never computed explicitly.
$\alpha_i$
A scalar — the dual coefficient (weight) attached to training example $i$. Collected into the vector $\boldsymbol\alpha \in \mathbb{R}^N$.
$\mathbf{w}^*$
The optimal weight vector, written as a combination of the training feature vectors — this is what the representer theorem guarantees.
$k(\mathbf{x}_i, \mathbf{x})$
A scalar — the kernel similarity between training point $i$ and a new input $\mathbf{x}$. The only thing the prediction actually needs.
$f^*(\mathbf{x})$
The predicted output: a weighted sum of similarities to the training points.

So we never need $\phi$ explicitly — we just need the kernel function and the dual coefficients $\boldsymbol\alpha \in \mathbb{R}^N$. For kernel ridge regression, $\boldsymbol\alpha = (\mathbf{K} + \lambda \mathbf{I})^{-1} \mathbf{y}$. For SVMs (Cortes & Vapnik 1995) you solve a quadratic program in $\boldsymbol\alpha$ with hinge loss and only the support vectors (the handful of training points lying on or inside the margin) end up nonzero.

§5Common kernels, and what they encode

  • Linear: $k(\mathbf{x}, \mathbf{x}') = \mathbf{x}^\top \mathbf{x}'$. No nonlinearity; useful as a baseline.
  • Polynomial: $k(\mathbf{x}, \mathbf{x}') = (\mathbf{x}^\top \mathbf{x}' + c)^p$. Implicit feature space is the space of monomials up to degree $p$.
  • RBF / Gaussian: $k(\mathbf{x}, \mathbf{x}') = \exp(-\norm{\mathbf{x} - \mathbf{x}'}^2 / (2\ell^2))$. Smooth bumps at every training point; the implicit feature space is infinite-dimensional. The length scale $\ell$ is the dominant hyperparameter — small $\ell$ overfits, large $\ell$ underfits.
  • Matérn: like RBF but tunably less smooth. Used in geostatistics and Gaussian processes (Chapter VIII of the comp-chem primer, if you have it nearby).

Kernels are also a clean way to build models for non-vector data. A kernel on graphs, on protein sequences, on text strings — anything where you can define a sensible inner-product-like similarity — gives you a linear model in that domain for free.

§6What killed kernels (and what didn't)

Kernel methods dominated 1995–2010. By 2015 deep nets had eaten their lunch on most tasks. Three reasons:

  1. Memory and compute scale poorly. Kernel methods need the $N \times N$ Gram matrix; training is $O(N^3)$ for a solve, prediction is $O(N)$ per query (you must touch every training point). Approximations exist — Nyström, random Fourier features (Rahimi & Recht 2007) — but they are workarounds, not solutions.
  2. Fixed feature maps don't learn. The kernel is chosen, not trained. For images, sequences, or any modality where the right features are not obvious, this becomes the bottleneck.
  3. Deep nets do learn features — and they kept getting better with scale. The 2012 ImageNet result (Chapter IV) was the moment the field's attention shifted permanently.

What did not die: the conceptual frame. Modern neural networks have a kernel interpretation in the infinite-width limit (Jacot et al. 2018, the Neural Tangent Kernel); Gaussian processes remain the right tool for small-data regression with calibrated uncertainty; the geometry of inner products and projections recurs in every chapter that follows. Kernels were the bridge between linear theory and the nonlinear practice — and they are still the right place to start when $N$ is small.
